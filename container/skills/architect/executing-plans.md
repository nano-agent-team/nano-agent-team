# Executing Plans

Use this skill when following a pre-existing implementation plan.

## Steps

1. **Read the full plan** — understand the entire plan before starting:
   - What is the end goal?
   - How many steps are there?
   - Are there any prerequisites?

2. **Validate prerequisites** — confirm everything needed is in place:
   - Required files/services exist
   - Dependencies are available
   - You have the necessary permissions

3. **Execute one step at a time** — do not combine steps unless explicitly instructed:
   - Complete each step fully
   - Verify the step result before moving to the next

4. **Track progress** — after each step note:
   - Status: done / skipped (with reason) / blocked
   - Any deviation from the plan and why

5. **Handle deviations** — if a step cannot be done as written:
   - Understand why it fails
   - Find the closest correct alternative
   - Document the deviation clearly

6. **Validate after completion** — run the verification steps from the plan:
   - Build, tests, acceptance criteria check

## Rules

- Do not skip steps without documenting the skip and reason
- If blocked, stop and report the blocker rather than guessing
- Deviations from the plan must be documented — do not silently change the approach
